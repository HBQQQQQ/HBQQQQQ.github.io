FLOPs && Parameters

发表于 2020-08-27 分类于 基础🐱

一、区别FLOPS和FLOPs：

FLPOS: floating point operations per second，每秒浮点运算次数，计算速度，衡量硬件性能指标
FLOPs: floating point operations，浮点运算数量，计算量，衡量模型复杂度
常用：1 GFLOPs = 10^9 FLOPs

二、卷积FLOPs计算：

🌰简单的例子：
image大小为5x5，卷积核大小为3x3，输入三通道，卷积核数量为3x9=27
那么一次卷积所需的运算量：(3x3)个乘法+(3x3-1)个加法 = 17
所以一共所需的运算量：17x3x9 = 459

🌲 = （计算每个输出特征值对应的乘法操作数+计算每个输出特征值对应的加法操作数）* 输出特征值的个数

🌟不考虑偏置项bias时：
(k^2 + k^2 -1) x H(out) x W(out) x C(in) x C(out)

🌟当考虑偏置项bias时加法的-1抵消：
2k^2 x H(out) x W(out) x C(in) x C(out)

三、参数量计算：

🌟不考虑偏置，参数量 = 每个卷积核的参数 * 核的数量（输出通道数）：
k^2 x C(in) x C(out)
🌟考虑偏置：
(C(in) x k^2 +1) x C(out)

数组中的重复数字

发表于 2020-08-01 分类于 剑指offer

在一个长度为n的数组里的所有数字都在0到n-1的范围内。 数组中某些数字是重复的，但不知道有几个数字是重复的。也不知道每个数字重复几次。请找出数组中任意一个重复的数字。 例如，如果输入长度为7的数组{2,3,1,0,2,5,3}，那么对应的输出是第一个重复的数字2。这里只返回是否有重复数字。

方法一、哈希法

🌟算法步骤：
1.开辟一个长度为n的vector<bool>, 初始化为false</bool>
2.遍历数组，第一次遇到的数据，对应置为true
3.如果再一次遇到已经置为true的数据，说明是重复的。返回即可。
⏰时间复杂度：O(N)
🌊空间复杂度：O(N)

方法二、in-place法

⏰时间复杂度：O(N)
🌊空间复杂度：O(1)

合并排序链表

发表于 2020-07-26 分类于 剑指offer

给两个非递减单链表l1, l2，合并为一个非递减的单链表。

方法一：迭代法

时间复杂度：O(m+n),m，n分别为两个单链表的长度
空间复杂度：O(1)

方法二：递归法

递归结束的条件：链表为空
递归方法：如果pHead1的所指节点值小于等于pHead2所指的结点值，那么pHead1后续节点和pHead2节点继续递归
时间复杂度：O(m+n)
空间复杂度：O(m+n),每一次递归，递归栈都会保存一个变量，最差情况会保存(m+n)个变量

反转链表

发表于 2020-07-25 更新于 2020-07-26 分类于 剑指offer

输入一个链表，反转链表后，输出新链表的表头。

🌟如果此类型的题出现在笔试中，如果内存要求不高，可以采用如下方法：
可以先用一个vector将单链表的指针都存起来，然后再构造链表。
🌲时间复杂度：O(n)
🌲空间复杂度：O(n), 用了一个vector来存单链表

🌟但是面试的时候，上一种解法当然不行。此题想考察的是：如何调整链表指针，来达到反转链表的目的。
初始化：3个指针
1）pre指针指向已经反转好的链表的最后一个节点，最开始没有反转，所以指向nullptr
2）cur指针指向待反转链表的第一个节点，最开始第一个节点待反转，所以指向head
3）nex指针指向待反转链表的第二个节点，目的是保存链表，因为cur改变指向后，后面的链表则失效了，所以需要保存
接下来，循环执行以下三个操作
1）nex = cur->next, 保存作用
2）cur->next = pre 未反转链表的第一个节点的下个指针指向已反转链表的最后一个节点
3）pre = cur， cur = nex; 指针后移，操作下一个未反转链表的第一个节点
循环条件，当然是cur != nullptr
循环结束后，cur当然为nullptr，所以返回pre，即为反转后的头结点
🌲时间复杂度：O(n), 遍历一次链表
🌲空间复杂度：O(1)

两个链表的第一个公共结点

发表于 2020-07-25 分类于 剑指offer

输入两个链表，找出它们的第一个公共结点。
🌟有公共结点的两个链表形状是y字型的，即公共结点之后两个链表是重合的。

方法1： 最容易想到的办法是用两个循环，遍历第一个链表，对每一个元素遍历第二个链表，找相同的结点。这种算法时间复杂度为O(mn)（假设链表长度分别为m和n），不适用这种方法。

方法2： 如果两个链表有公共结点，那么从公共结点之后链表便重合了，例如1-2-3-4-5 和 3-9-2-3-4-5，尾部对齐后，只要找到2的位置就是第一个公共结点。链表长度差为1，两指针分别指向两头结点，那么第一个指针先走掉相差的1个结点到达9，第二个链表再开始走，就能同步比较两个链表了，这时往后遍历找到2即可。这种算法时间复杂度为O(m+n)。

方法3： 我们使用两个指针 node1，node2 分别指向两个链表 headA，headB 的头结点，然后同时分别逐结点遍历，当 node1 到达链表 headA 的末尾时，重新定位到链表 headB 的头结点；当 node2 到达链表 headB 的末尾时，重新定位到链表 headA 的头结点。这样，当它们相遇时，所指向的结点就是第一个公共结点。这种算法时间复杂度为O(m+n)，空间复杂度O(1)。

链表中环的入口结点

发表于 2020-07-22 更新于 2020-07-24 分类于 剑指offer

给一个链表，若其中包含环，请找出该链表的环的入口结点，否则，输出null。
🌟首先判断链表是否有环：
设置快慢指针fast和slow，fast每次走两步，slow每次走一步，假如有环，两者一定会相遇。
🌟然后判断环的入口结点：
两个指针分别从链表头和相遇点继续出发，每次走一步，最后一定在环的入口相遇。
证明：
设链表头到环的距离—a
环入口到相遇点距离—b
相遇点到环入口距离—c
则相遇时：
fast走过的路程=a+k(b+c)+b
slow走过的路程=a+n(b+c)+b
因为fast=2*slow
所以可得a=(k-2n-1)(b+c)+c。即链表头到环入口的距离=相遇点到环入口的距离+(k-2n-1)的环长度，所以两个指针分别从链表头和相遇点出发，最后一定相遇于环入口。
